Yes. I would treat this as an interactive builtin, not an ordinary builtin.

The builtin should not itself know about SwiftUI menus. It should ask an interface object:

protocol ProgramInteraction {
    func choosePerson(matching pattern: String) throws -> Person?
}

Then Program gets one more dependency:

let interaction: ProgramInteraction?

The builtin becomes conceptually:

	func builtinGetPerson(_ args: [ParsedExpr]) throws -> ProgramValue {
    let pattern = try evaluateString(args[0], errMsg: "getperson: arg must be a string")

    guard let interaction else {
        throw RuntimeError("getperson: no interactive UI available", line: args[0].line)
    }

    guard let person = try interaction.choosePerson(matching: pattern) else {
        return .null
    }

    return .person(person)
}

But here is the hard part: SwiftUI selection is asynchronous. The old LifeLines version could block in curses. In SwiftUI, you probably do not want the interpreter freezing while waiting for a panel.

So there are two design paths.

The simple first implementation: make getperson("wetmore") search and return the first/best match, or throw if ambiguous. This tests the builtin plumbing but does not solve UI.

The real implementation: make the interpreter capable of suspending at an interactive builtin, show a selection panel, then resume when the user chooses. That is a bigger project.

A very practical intermediate approach:

1. Add a ProgramPage side panel/modal for “pending person request.”
2. When getperson() runs, it throws or returns a special result like .needsPersonSelection(request).
3. ProgramModel stores that request and shows the picker.
4. After the user chooses, ProgramModel resumes or reruns with the selected value.

Given your current interpreter is synchronous, I would not try full suspend/resume immediately.

My PersonSelectionView has this function:

    /// Build array of PersonMatch objects that match query; sort them by name, and
    /// user event dates to improve the order.
    private func doNameSearch() {
        guard let database = model.database else { return }
        let index = database.recordIndex

        results = database.persons(withName: namePattern)
            .map { PersonMatch(id: $0.key, person: $0) }
            .sorted { lhs, rhs in
                switch lhs.person.compare(to: rhs.person, in: index) {
                case .orderedAscending:
                    return true
                case .orderedDescending:
                    return false
                case .orderedSame:
                    return lhs.id < rhs.id
                }
            }
    }

It calls Database.persons(withName: namePattern):

extension Database {
    /// Return array of all persons with names that match a pattern.
    public func persons(withName pattern: String) -> [Person] {
        personKeys(forName: pattern).compactMap { recordIndex.person(for: $0) }
    }
}

which calls:

extension Database {

    /// Return array of record keys of all persons with names that match a pattern.
    public func personKeys(forName pattern: String) -> [RecordKey] {

        var matchingKeys: [String] = []
        let nameKey = nameKey(value: pattern) // Name key of name pattern.
        guard let recordKeys = nameIndex.index[nameKey] else { return [] }  // Persons keys that match nameKey.
        let squeezedPattern: [String] = squeeze(pattern) // Prepare pattern for matching.
        // Filter candidates based on exactMatch logic.
        for recordKey in recordKeys {
            if let person = recordIndex.person(for: recordKey) {
                for nameValue in person.nameValues {
                    let squeezedPersonName = squeeze(nameValue)
                    if exactMatch(partial: squeezedPattern, complete: squeezedPersonName) {
                        matchingKeys.append(recordKey)
                        break // No need to check other names of this Person.
                    }
                }
            }
        }
        return matchingKeys
    }
}
