Practical first version

1. Implement an explicit interpreter continuation state.
2. Run the program in an async context and make chooseperson await the picker result.
3. Throw a special “need input” signal and later rerun/replay with stored answer.

@MainActor
final class ProgramModel: ProgramInteraction {
    var pendingPersonChoices: [Person] = []
    var isShowingPersonPicker = false

    private var personContinuation: CheckedContinuation<Person?, Never>?

    func choosePerson(from candidates: [Person], prompt: String) async -> Person? {
        pendingPersonChoices = candidates
        isShowingPersonPicker = true

        return await withCheckedContinuation { continuation in
            personContinuation = continuation
        }

    func choose(_ person: Person?) {
        isShowingPersonPicker = false
        personContinuation?.resume(returning: person)
        personContinuation = nil
    }
}

→ builtin returns .person or .null
→ interpreter continues

1. Can synchronous interpreter code “wait for SwiftUI” like ncurses?

Not cleanly. In SwiftUI, you should not block the main thread waiting for UI input. The ncurses model was:

call getperson()
block until user chooses
continue

The SwiftUI model should be:

call chooseperson()
await user choice
UI remains alive
continue

So yes: async/await is the right tool. It gives you the logical feel of blocking, without freezing the UI.

⸻

2. If a method is async, must callers use await?

Yes.

func choosePerson() async -> Person?

must be called as:

let person = await choosePerson()

And the caller must itself be async, unless it starts a new task:

Task {
    let person = await choosePerson()
}

So the “async infection” travels upward through the call chain.

But this part is important:

Functions called by async functions do not need to become async unless they themselves suspend.

So an async function can call ordinary synchronous helpers freely:

func evaluate(...) async throws -> ProgramValue {
    let name = person.displayName()   // ordinary sync call, fine
    return .string(name)

    diagnostics = []
    output.clear()

        parsedProgram: parsedProgram,

        interaction: self

        let result = try await program.interpretProgram()
        output.text = displayableOutput(buffer.text)
        // handle result if needed
    } catch let error as RuntimeError {
        output.text = displayableOutput(buffer.text)
        diagnostics = [Diagnostic(
            message: error.message,
            line: error.line > 0 ? error.line : nil
        )]
    } catch {
        output.text = displayableOutput(buffer.text)
        diagnostics = [Diagnostic(message: String(describing: error), line: nil)]
    }
}
